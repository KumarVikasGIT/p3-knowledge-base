# A share link is showing an error

A few distinct things can cause this — the fix depends on which.

## The gallery was set to private

If the gallery owner turned off link-only sharing, the link stops working immediately for everyone, including people who had it before. There's no error message that distinguishes this from other causes on your end — if you suspect this, ask the gallery owner directly whether the gallery is still set to link-only.

## The link was regenerated

If the owner generated a new share link, the old one stops working — this is intentional, since regenerating is how an owner replaces a link they no longer want active. Ask for the current link if you suspect this happened.

## The link looks incomplete or was altered

Share links contain an important part after a `#` symbol — if that part is missing, cut off, or the link was copied incorrectly (for example, some messaging apps occasionally mishandle long links with special characters), the link won't work even though the rest of the address looks correct. Try asking for the link to be re-sent, ideally copied directly rather than retyped, and check that the full link — including everything after the `#` — made it through.

## You don't have permission, or it's a client link tied to someone else

Some links are tied to a specific delivery and may not work if opened in the wrong context. If you believe you should have access and nothing here resolves it, check with whoever shared the link with you.

## If it's your own gallery and you can't get the link to work
